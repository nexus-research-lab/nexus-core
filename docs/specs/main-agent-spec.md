# 主智能体规范

## 1. 文档目标

本文档定义 Nexus 主智能体的身份、入口和职责边界。

当前仓库已经切换到 Go 后端，主智能体规范只描述当前有效实现。

## 2. 主智能体是什么

主智能体是系统保留的默认 agent。

它承担三类职责：

- 默认入口承接
- 平台级协作编排
- 系统能力托管

它不是普通成员 agent 的一个特殊别名。

## 3. 真相源

### 3.1 身份真相源

主智能体身份以服务端配置为准：

- `internal/config`
- `/nexus/v1/runtime/options`

前端只消费后端下发的 `default_agent_id`，不再维护第二份默认值。

### 3.2 Workspace 真相源

主智能体拥有固定 workspace：

- `workspace/<default_agent_id>/`

### 3.3 路由真相源

主智能体的默认入口路由仍然走标准 DM / room 语义，不单独创造新的会话协议。

## 4. 当前职责边界

### 4.1 主智能体负责

- 默认入口对话
- 平台级组织动作
- 作为系统保留 agent 存在

### 4.2 主智能体不负责

- 代替所有专业 agent 执行具体协作
- 充当普通 room 成员模板
- 让前端通过硬编码名字识别

## 5. 初始化规则

系统启动后必须保证：

- 主智能体记录存在
- 主智能体 workspace 可用
- 相关默认配置能通过 runtime options 下发给前端

## 6. 前端规则

- 前端通过 `/nexus/v1/runtime/options` 获取 `default_agent_id`
- launcher 和 `/app` 不允许手写第二份默认主智能体 ID
- 普通成员目录可以过滤主智能体，但核心 agent cache 必须能解析到它

## 7. 技能与系统能力

主智能体可以拥有平台级内部技能，但这些技能不应自动等价成“公开能力市场里的普通技能”。

区分原则：

- 平台内部编排能力：系统托管
- 普通协作能力：按公开 skill / capability 机制管理

## 8. 禁止项

- 硬编码 `"nexus"` 或 `"main"` 作为业务判断真相源
- 把主智能体当普通成员模板使用
- 让前端自己推断主智能体身份

## 9. 一句话总结

主智能体是系统保留 agent，不是 UI 文案，不是前端默认值，也不是普通成员的超集。
